# Internationalization (i18n) Implementation Plan

This document outlines the effort and strategy for internationalizing the Billiards UI for Korea (ko), Japan (ja), and Vietnam (vi).

## 1. Audit Summary
User-facing strings identified for localization:
- **Static UI (index.html)**: Buttons (hit, send, close, accept, decline), Tooltips (aria-labels), Placeholders (chat input).
- **Dynamic UI (TypeScript)**: Notification titles and subtexts (Foul, Game Over, Concede), Menu items, Score updates.
- **Help Content (help.html)**: Control guides, Aiming instructions.
- **Physical Constants**: Labels (R, m, e, mu, etc.).

## 2. Localization Strategy

### Infrastructure (src/i18n/)
- **TranslationService.ts**: A singleton service to manage the current locale.
  - Detection: Prioritize `lang` URL parameter, then `localStorage`, then `navigator.language`.
  - Methods: `t(key: string, variables?: object)` for translation lookup.
- **Translations Interface**: A TypeScript interface to ensure consistency across language files.

### Language Files
Create `src/i18n/locales/`:
- `en.ts` (Reference)
- `ko.ts` (Korean)
- `ja.ts` (Japanese)
- `vi.ts` (Vietnamese)

## 3. Implementation Plan

### Phase 1: Core Service & English Baseline
1.  Implement `TranslationService`.
2.  Extract all strings from TypeScript files into `en.ts`.
3.  Replace hardcoded strings in `Notification.ts`, `Menu.ts`, and Rules (`snooker.ts`, `nineball.ts`, etc.) with `TranslationService.t()`.

### Phase 2: index.html Localization
1.  Add `data-i18n` attributes to HTML elements in `dist/index.html`.
2.  Implement a DOM translation utility that runs on page load to update `innerText`, `placeholder`, and `aria-label` based on `data-i18n` keys.

### Phase 3: Language-Specific Support
1.  **Korea (ko)**: Use standard terminology (e.g., "3쿠션" for Three-Cushion). Ensure fonts support Hangul.
2.  **Japan (ja)**: Use Katakana for technical terms (e.g., "スヌーカー" for Snooker). Validate vertical space for Kanji/Kana.
3.  **Vietnam (vi)**: Support accented characters (e.g., "Bida 3 băng"). Adjust layouts for longer Vietnamese strings.

### Phase 4: help.html and Overlays
1.  Pass the locale to the help iframe via query parameters (`help.html?lang=ko`).
2.  Implement simple JS in `help.html` to swap content blocks or labels based on the parameter.

## 4. Verification
- **Automated Tests**: Unit tests for `TranslationService` to ensure correct fallback to English.
- **Visual Audit**: Verify UI layout in all languages. Check for "text overflow" in buttons and notifications.
- **Font Rendering**: Confirm all characters (Hangul, Kanji/Kana, Vietnamese diacritics) render correctly across browsers.
