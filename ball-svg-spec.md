# Ball SVG Comment Emoji — Spec

## Summary

Replace the hardcoded inline SVG in `dist/index.html`'s comment menu with a code-generated approach. Add 4 ball-SVG buttons (N/S/E/W dot positions) driven by a `data-angle` HTML attribute, with SVG generated by a `ballSvg()` function in `chat.ts`.

---

## Motivation

- Remove inline SVG clutter from `dist/index.html`
- Enable multiple directional variants without copy-pasting SVG markup
- Keep SVG generation in one place (TypeScript) so it's easy to modify/tune
- Minimize code intrusion — leverage existing `comment.ts` button-click → `chat.ts` render pipeline

---

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Home for `ballSvg()` | `src/view/chat.ts` | chat.ts already understands SVG tags; avoids a new file |
| How buttons declare intent | `data-angle` attribute | Declarative, HTML-readable, easy to parse |
| Button count | Add 4 new (total 16) | Preserves all existing emoji |
| Button placement | Prepend to start of menu | New balls appear first in the 3-column grid |
| Hydration strategy | Opt-in: only `[data-angle]` buttons get SVG | Non-ball buttons remain untouched |
| Chat message content | Send full SVG markup | Consistent with existing inline SVG button behavior |
| JS-off fallback | None | Game requires JS; empty buttons are acceptable |
| Dot position offset | 5.0px from center | Gives ~0.7px breathing room inside the 7.2-radius white circle |
| SVG size | 20×20, viewBox 0 0 20 20 | Matches existing inline SVG |
| Colors | All four: white circle, black stroke, red dot | Only position differs — simplicity |
| Testing | No separate unit test | Integration covered by existing comment.spec.ts |

---

## Math

- SVG center: `(10, 10)`
- White circle: `cx=10, cy=10, r=7.2`
- Red dot: `r=1.5`, offset radius `r=5.0`
- Angle convention: 0°=N ↑, 90°=E →, 180°=S ↓, 270°=W ←

```
cx = 10 + 5.0 × sin(angle)
cy = 10 − 5.0 × cos(angle)
```

| Angle | Direction | cx     | cy   |
|-------|-----------|--------|------|
| 0°    | N (top)   | 10.0   | 5.0  |
| 90°   | E (right) | 15.0   | 10.0 |
| 180°  | S (bottom)| 10.0   | 15.0 |
| 270°  | W (left)  | 5.0    | 10.0 |

Clearance check: dot edge at radius 5.0 + 1.5 = 6.5; white circle edge at 7.2; gap = 0.7px.

### 2. `src/view/chat.ts` — Add `ballSvg()` function

Add an exported function:

```typescript
/**
 * Generate SVG markup for a ball icon with a red dot at the given angle.
 * @param angleDeg - 0=N ↑, 90=E →, 180=S ↓, 270=W ←
 * @returns SVG markup string
 */
export function ballSvg(angleDeg: number): string {
  const rad = (angleDeg * Math.PI) / 180
  const r = 5.0  // dot offset from center
  const cx = (10 + r * Math.sin(rad)).toFixed(1)
  const cy = (10 - r * Math.cos(rad)).toFixed(1)
  return (
    `<svg width="20" height="20" viewBox="0 0 20 20" style="vertical-align:middle">` +
    `<circle cx="10" cy="10" r="7.2" fill="white" stroke="black" stroke-width="1"/>` +
    `<circle cx="${cx}" cy="${cy}" r="1.5" fill="red"/>` +
    `</svg>`
  )
}
```

### 3. `src/view/comment.ts` — Hydrate data-angle buttons

In the constructor, after getting `this.menu`, add:

```typescript
// Hydrate ball-SVG buttons
this.menu.querySelectorAll<HTMLButtonElement>('.comment-emoji[data-angle]')
  .forEach(btn => {
    const angle = parseInt(btn.dataset.angle ?? '0', 10)
    btn.innerHTML = ballSvg(angle)
  })
```

Import `ballSvg` from `./chat`.

## Behavior Walkthrough

1. **Page load**: HTML contains `<button class="comment-emoji" data-angle="0"></button>` (empty, no SVG).
2. **JS init**: `Comment` constructor calls `querySelectorAll('[data-angle]')`, finds 4 buttons, sets each `innerHTML = ballSvg(angle)`.
3. **User clicks ball button**: `comment.ts` click handler reads `btn.innerHTML` (now the SVG string), sends it via `container.sendChat()`.
4. **Chat receives SVG**: `chat.ts` `showMessage()` detects `<` in the string, sanitizes, allows `svg`/`circle` tags, appends to chat output.
5. **Remote player sees SVG**: Same code path — SVG string arrives, is sanitized, rendered in their chat.

---

## Non-Goals

- No CSS changes needed (existing `.comment-emoji` styles suffice)
- No new npm dependencies
- No changes to the chat sanitization logic (`allowedTags` already includes `svg` and `circle`)
- No changes to `sendChat()` or the network protocol (SVG is sent as a string like any other emoji)

---

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Math bug gives wrong dot position | Manual verification against angle table above; visual check in browser |
| Angular convention confusion (sin/cos swap) | Explicitly documented: 0°=N, 90°=E. uses `sin` for x, `cos` for y |
| `data-angle` parsed as NaN | `parseInt(..., 10)` with `?? '0'` fallback; logs to console? (TBD) |
| Future dev adds non-numeric data-angle | Hydration silently produces garbage SVG if angle is NaN → `toFixed(1)` on NaN = "NaN". Could add `isNaN` guard |
